# Multi-Layer Perceptron - Fashion-MNIST Classification

**CS3807 – Deep Learning Laboratory, Experiment 2 - Multi-Layer Perceptron**

A Multi-Layer Perceptron built with TensorFlow/Keras to classify clothing images into 10
categories, using the Fashion-MNIST dataset, followed by automated hyperparameter
optimization.

## Overview

This project covers the full pipeline end to end:

- Dataset exploration (sample images, class distribution)
- Preprocessing (flattening $28\times28$ images to 784-dim vectors, normalization, one-hot
  encoding)
- A baseline MLP built in Keras - Dense(128, ReLU) $\to$ Dense(64, ReLU) $\to$ Dense(10,
  Softmax), with full epoch-by-epoch training logs
- Evaluation (accuracy, precision, recall, F1-score, confusion matrix, classification report)
- Automated hyperparameter optimization with `RandomizedSearchCV` + SciKeras
  (`KerasClassifier`), 5-fold cross-validation over hidden layers, hidden neurons, learning
  rate, batch size, epochs, optimizer, activation function, and dropout rate
- Retraining and evaluating the optimized model on the full training set
- A baseline vs. optimized comparison, with a second confusion matrix for the optimized model

## Results

| Metric    | Baseline Model | Optimized Model |
|-----------|-----------------|-------------------|
| Accuracy  | 88.36%           | 88.71%             |
| Precision | 88.53%           | 88.71%             |
| Recall    | 88.36%           | 88.71%             |
| F1-score  | 88.39%           | 88.68%             |

Baseline trained for 20 epochs at batch size 32 (Adam, lr = 0.001). Optimized model uses the
best configuration found by the search - 1 hidden layer, 128 neurons, sigmoid activation, no
dropout, Adam (lr = 0.001), batch size 128, 30 epochs (5-fold CV accuracy 85.79%) - retrained
on the full 60,000-image training set and evaluated on the 10,000-image test set.

## Dataset

**Fashion-MNIST Dataset** — Zalando Research, fetched programmatically via
`tensorflow.keras.datasets.fashion_mnist`, so no manual download is required.

- 70,000 grayscale images total (60,000 train / 10,000 test), $28 \times 28$ pixels,
  10 clothing classes
- Classes perfectly balanced (6,000 training images per class)
- No missing values

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/<your-repo-name>.git
cd <your-repo-name>
```

### 2. Set up a virtual environment (recommended)

```bash
python3 -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Run the notebook

```bash
jupyter notebook deep_learning_lab_2.ipynb
```

or open it directly in [Google Colab](https://colab.research.google.com/) (GPU runtime
recommended - the hyperparameter search trains around 75 models) and run all cells top to
bottom. The notebook fetches the dataset itself - nothing needs to be downloaded by hand.

## Building the Report

The LaTeX report lives in `report/main.tex` and pulls in the figures from `report/figures/`.
The figures are already PDF (not EPS), so no `-shell-escape` flag is required.

1. Upload `report/main.tex` and everything in `report/figures/` into an
   [Overleaf](https://www.overleaf.com/) project (same folder level - keep the `figures/`
   subfolder as is).
2. Compile with pdfLaTeX.

Or locally:

```bash
cd report
pdflatex main.tex
pdflatex main.tex
```
